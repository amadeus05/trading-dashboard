
## Страница проверки здоровья (Health)

Дашборд включает специальную страницу для мониторинга состояния всех компонентов системы.

### URL

```
http://localhost:5174/health
```

### Что проверяется

| Компонент | Описание проверки |
|-----------|-------------------|
| **Backend API** | Доступность API endpoints (через `/api/metrics`) |
| **База данных** | Соединение с БД (определяется через успешность API запроса) |
| **Frontend** | Статус интерфейса (всегда "Работает") |
| **Окружение** | Режим работы (development/production/demo) |

### Внешний вид страницы

Страница состоит из 4 карточек статуса:

1. **Backend API** — иконка сервера
   - 🟢 Зеленая — API отвечает корректно
   - 🔴 Красная — не удалось подключиться
   - 🟡 Желтая — идет проверка

2. **База данных** — иконка базы данных
   - 🟢 Зеленая — соединение с БД установлено
   - 🔴 Красная — проблемы с подключением

3. **Frontend** — иконка щита
   - 🔵 Индиго — Vue.js + Vite + Tailwind работают

4. **Окружение** — иконка сигнала
   - 🟣 Фиолетовая — показывает режим (DEMO/PROD/DEV)
   - Бейдж "DEMO" если включен демо-режим

### Функциональность

- **Автоматическая проверка** каждые 30 секунд
- **Ручная проверка** — кнопка "Проверить сейчас"
- **Время отклика** — показывается в миллисекундах
- **Время последней проверки** — timestamp
- **Общий статус** — индикатор "Все системы работают" или "Обнаружены проблемы"
- **Системная информация** — версии, время сборки, режим окружения

### Поведение в разных режимах

| Режим | Backend | Database |
|-------|---------|----------|
| **DEMO** (VITE_DEMO_MODE=true) | Всегда "Здоров" (заглушка) | Всегда "Здоров" |
| **Реальный** с работающим backend | Проверяет реальный API | Определяется по API ответу |
| **Реальный** с недоступным backend | "Недоступен" (красный) | "Недоступен" |

### Технические детали

- Использует `dashboardApi.getMetrics()` для проверки health
- Интервал проверки: **30 секунд** (константа `CHECK_INTERVAL`)
- В демо-режиме проверка эмулируется (всегда успешная)
- При ошибке показывается красный статус с описанием

### Пример использования

Страницу Health можно использовать для:
- 🔍 Мониторинга состояния trading-системы
- 🚨 Быстрой диагностики проблем подключения
- 📊 Отображения в NOC/мониторинге на TV
- 🔄 Проверки после деплоя
